# BPIC-17 Analysis (IN0012 Individual Assignment)

Process-mining analysis of the BPI Challenge 2017 event log for the course
Business Process Prediction, Simulation, and Optimization (TUM, SS 2026).

This is a **code-only** repository: the five analysis notebooks, the
reusable src/ modules and the BPMN model. It is
self-contained except for the event log, which is too large to
redistribute and is downloaded from its original source (step 2). Every
analysis output (results/ (tables and figures) and models/
(discovered nets)) is regenerated by running the notebooks and is not
committed.

### Repository layout
* data/             (event log goes here (only .gitkeep is tracked))
* notebooks/        (analysis notebooks 01–05 (run in order))
* src/              (reusable modules (seeds, preprocessing, metrics))
* Signavio export/  (final BPMN model (the deliverable)), (v4_replay.bpmn is a hard input of notebook 03)

Generated, not committed (git-ignored, recreated by the notebooks):
* results/, models/, and the downloaded data/ contents.

## Reproduction

### 1. Install the environment

**bash:** 
pipenv install

Python 3.14. All dependencies are pinned with hashes in Pipfile.lock.

### 2. Download the event log

Download **BPI Challenge 2017.xes** from the official 4TU.ResearchData
record and place it in **data/**:

* van Dongen, B. (2017). BPI Challenge 2017. 4TU.ResearchData.
* dataset page: https://data.4tu.nl/articles/dataset/BPI_Challenge_2017/12696884

Notebook 01 caches a filtered pandas pickle next to it so notebooks 02–05 do not
re-parse the raw XES.

### 3. Run the notebooks in order

**bash:**

pipenv run jupyter nbconvert --to notebook --execute --inplace \
  notebooks/01_eventlog_analysis.ipynb \
  notebooks/02_discovery.ipynb \
  notebooks/03_conformance.ipynb \
  notebooks/04_resource_analysis.ipynb \
  notebooks/05_decision_mining.ipynb


(or open them in pipenv run jupyter notebook and run 01 -> 05 top to
bottom). This recreates results/ and models/.

Notebook 03 reads the final BPMN Signavio export/v4_replay.bpmn (included in the repository) 
to compute the enriched-vs-auto quality comparison, so no extra input is needed.

All random seeds are set centrally in [src/seeds.py](src/seeds.py)
(**RANDOM_SEED = 42**).

### Notebooks

| # | Notebook | Analysis |
| :--- | :--- | :--- |
| 01 | 01_eventlog_analysis.ipynb | Event-log statistics, variant Pareto, filter impact |
| 02 | 02_discovery.ipynb | Process discovery (Alpha / Heuristics / Inductive) |
| 03 | 03_conformance.ipynb | Conformance checking + final model + enriched-BPMN comparison |
| 04 | 04_resource_analysis.ipynb | BPMN lane mapping + social-network mining |
| 05 | 05_decision_mining.ipynb | Decision mining at the Application-Outcome gateway |
